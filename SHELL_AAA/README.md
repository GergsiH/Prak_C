# Проект: Реализация собственного командного интерпретатора (Shell)

В этом проекте я разработала упрощённую версию командного интерпретатора, аналогичного Unix shell, с поддержкой основных функций для выполнения команд и управления процессами.

## Что я реализовала

### 1. Базовый цикл чтения-выполнения
Создала основной цикл shell, который принимает команды пользователя, парсит их и выполняет. Реализовала встроенные команды (cd, exit, help) и возможность запуска внешних исполняемых файлов через поиск в PATH.

### 2. Управление процессами и пайплайны
Добавила поддержку фонового выполнения процессов с управлением через сигналы. Реализовала пайплайны (конвейеры) для перенаправления вывода одной команды на вход другой, включая обработку нескольких последовательных пайпов.

### 3. Перенаправление ввода-вывода и работа с файлами
Настроила перенаправление стандартных потоков (stdin, stdout, stderr) в файлы и из файлов с поддержкой режимов добавления в конец файла. Реализовала подстановку переменных окружения и обработку специальных символов, включая wildcards.

### 4. Обработка сигналов и управление заданиями
Добавила корректную обработку сигналов (Ctrl+C, Ctrl+Z, Ctrl+\) с сохранением контроля над дочерними процессами. Реализовала систему управления заданиями (jobs) с командами fg, bg, jobs и возможностью возобновления приостановленных процессов.

## Особенности реализации и технологии
Проект написан на C с использованием системных вызовов Unix (fork, exec, wait, pipe, dup2). Для разбора команд применяется рекурсивный парсинг с построением абстрактного синтаксического дерева. Все процессы организованы в группы для корректной обработки сигналов, реализовано логирование ошибок и диагностика проблем с выполнением команд. Shell поддерживает историю команд с возможностью навигации и редактирования, имеет конфигурируемые настройки через файл конфигурации и цветовой вывод для улучшения пользовательского опыта.
